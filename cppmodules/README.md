# 高频交易回测引擎 (HFT Backtest Engine - C++)

本项目是一个基于事件驱动（Event-Driven）的低延迟高频量化回测引擎，完全使用 C++ 编写。该引擎专为评估做市商（Market Maker）策略而设计，内置了精细的网络延迟模拟、订单簿排队位置仿真以及基于微观结构的动态定价模型。

## 核心特性

* **高精度事件模拟**：利用最小堆（`std::priority_queue`）严格模拟订单在网络传输过程中的微秒级延迟（Latency）。
* **订单簿状态机**：维护本地的完整 L2 盘口状态（`book_bids` / `book_asks`），并能根据时间戳自动清理“幽灵订单”。
* **队列位置仿真**：引入 `queue_ahead` 概念，真实模拟限价单在交易所撮合引擎中的排队机制，避免过度乐观的成交预期。
* **微观结构定价策略**：
  * **订单簿不平衡指标 (OBI)**：实时感知盘口买卖力量对比，设定防守阈值防止被毒性流量（Toxic Flow）击穿。
  * **库存偏斜定价 (Inventory Skewing)**：通过风险厌恶系数（Gamma）动态调整基准价格，控制敞口风险。
* **双边费率支持**：原生支持 Maker 返佣与 Taker 惩罚机制。

## 模块说明

本项目包含以下核心源文件：

* **`Types.h`**: 定义了系统底层的数据结构。包含严格内存对齐（`#pragma pack(1)`）的 `TickEvent`，以及掩码定义、订单状态记录等。
* **`HighFreqBacktester.h` & `.cpp`**: 引擎的心脏。包含行情更新、策略下单/撤单逻辑、延迟事件分发以及模拟撮合核心逻辑。
* **`Reporter.h` & `.cpp`**: 负责性能评估与数据持久化。计算夏普比率（Sharpe）、卡玛比率（Calmar）、最大回撤等关键财务指标，并导出 CSV 记录。
* **`main.cpp`**: 程序的入口点。负责按天序贯读取底层二进制行情数据（`.bin`），驱动引擎运行，并调用报告模块。

## 编译与运行指南 

### 1. 环境依赖
* 支持 C++11 或更高版本的编译器（如 GCC, Clang, MSVC）。
* 已通过 Python 数据清洗管道生成好对应的 `.bin` 结构化二进制行情数据。

### 2. 路径配置
在编译前，请打开 `main.cpp`，定位到文件流读取部分，确保底层行情数据的路径与实际存放位置一致：
```cpp
oss << R"(C:\Users\Administrator\Desktop\cpp_market_data\market_data_2026_01_)"
    << std::setw(2) << std::setfill('0') << day 
    << ".bin";
```

### 3. 编译命令 (以 GCC 为例)
在包含所有源文件的目录下打开终端，执行以下命令构建可执行文件：

```bash
g++ -O3 -std=c++11 main.cpp HighFreqBacktester.cpp Reporter.cpp -o hft_engine
```
> 注：务必开启 `-O3` 优化标志，以发挥 C++ 的最佳执行性能。

### 4. 运行引擎
```bash
./hft_engine
```
引擎将自动逐日加载数据，并在终端输出每日的 Tick 处理数量与当前账户净值。

## 输出文件

运行结束后，系统将在当前目录下生成两个详细的数据文件，可供后续的 Python 脚本进行可视化分析：

* **`equity_curve.csv`**: 资金曲线日志，记录指定时间间隔内的账户总资产变化。
* **`trade_history.csv`**: 详尽的逐笔交易流水，包含时间戳、买卖方向、成交价格、数量、发生的手续费/返佣以及成交后的即时持仓量。
