# 高频交易回测系统 (HFT Backtester) - Data Pipeline

本项目包含一套专为低延迟、高频量化回测设计的数据 ETL（提取、转换、加载）与可视化流水线。主要负责将 OKX 交易所的原始 L2 订单簿 (Orderbook) 和逐笔成交 (Trades) 数据转化为底层回测引擎可直接进行内存映射 (Memory Mapping) 的紧凑二进制格式。

## 架构概述

整个流水线的设计遵循**极致性能**与**严格对齐**的原则。为了配合低延迟引擎的解析速度，所有行情事件均被统一封装为 32 字节的结构化实体，并利用位运算 (Bitmask) 标记事件类型与买卖方向。

数据流向如下：
`原始压缩数据 (.tar.gz / .zip)` -> `高并发清洗映射 (.npz)` -> `时间戳排序合并` -> `底层结构体对齐的二进制流 (.bin)` -> `回测结果可视化 (.png)`

## 核心数据结构设计

为了实现与底层系统内存布局的完美对齐，所有 Python 脚本均采用统一的 NumPy 结构化数组类型（对应 C++ 中的 `#pragma pack(1)` 结构体）：

```python
STRUCT_DTYPE = np.dtype([
    ('ev', 'u8'),  # 8字节无符号整数：事件掩码 (Event Mask)
    ('ts', 'i8'),  # 8字节带符号整数：微秒级时间戳 (Timestamp)
    ('px', 'f8'),  # 8字节浮点数：价格 (Price)
    ('qty', 'f8')  # 8字节浮点数：数量 (Quantity)
])
```

**事件掩码 (Event Mask) 定义：**
* `DEPTH_EVENT` (第28位) / `TRADE_EVENT` (第31位)
* `BUY_SIDE` (末位置1) / `SELL_SIDE` (末位置2)

## 模块详解

### 1. 数据清洗与编译模块
* **`merge.py` (核心枢纽)**
    * **功能**：单步完成 L2 订单簿与逐笔成交数据的联合解析、清洗、时间戳对齐排序，并直接输出底层引擎所需的 `.bin` 二进制文件。
    * **特性**：内置严格的错误捕获与熔断机制（容忍一定数量的 JSON 解析错误，超限则阻断），确保输入回测引擎的数据质量。采用多进程并发处理单日数据。
* **`extractob.py`**
    * **功能**：专门针对 L2 Orderbook `.tar.gz` 文件的解析脚本。利用 `orjson` 极速读取 JSON 行，截取指定深度的盘口数据，并序列化为 `.npz` 格式缓存。
* **`extracttrades.py`**
    * **功能**：专门针对 Trades 逐笔成交 `.zip` 文件的解析脚本。处理时区偏移（自动修正 +8 小时至 UTC），通过 Pandas 分块读取内存并向量化生成掩码，输出 `.npz`。

### 2. 回测分析模块
* **`plot.py`**
    * **功能**：读取回测引擎输出的 `equity_curve.csv` 和 `trade_history.csv`，生成专业的策略表现分析图。
    * **输出内容**：包含双 Y 轴资金曲线、精确映射到价格曲线上具体的买卖成交点（散点图），以及动态最大回撤 (Drawdown) 区域图。

### 3. 数据探查工具
* **`opendata.py`**
    * **功能**：大文件二进制探测器。用于安全读取并以十六进制/强制 UTF-8 格式打印未知大文件（如 `.data`）的头部字节，防止直接使用文本编辑器打开导致内存崩溃。
* **`readnpz.py`**
    * **功能**：`.npz` 数据校验工具。通过 `mmap_mode='r'` 只读内存映射技术，极速加载并校验清洗后数据的字段正确性。

## 使用指南

### 环境依赖
确保运行环境中已安装以下核心库：
```bash
pip install numpy pandas orjson matplotlib
```

### 执行流程
1.  **路径配置**：在运行前，请进入 `merge.py` (或分步执行的 `extractob.py` / `extracttrades.py`)，修改 `OB_DIR` 和 `TR_DIR` 为原始数据的实际存放路径，并配置输出目录。
2.  **生成底层数据**：
    运行主编译脚本：
    ```bash
    python merge.py
    ```
    脚本将启动多进程引擎，每日生成一个 `.bin` 文件（例如 `market_data_2026_01_01.bin`），该文件可直接交由底层回测系统读取。
3.  **生成回测报告**：
    待底层系统运行完毕并输出 CSV 记录后，在同级目录下执行：
    ```bash
    python plot.py
    ```
    即可生成 `backtest_report.png`。

## 注意事项
* 内存管理：高频原始数据解压后体积巨大。数据清洗脚本已使用 `gc.collect()` 显式回收内存，但在设定 `ProcessPoolExecutor` 的 `max_workers` 时，仍需根据运行机器的物理内存进行合理配置（建议单核预留 4GB 内存）。
* 时间戳基准：所有处理后的时间戳统一转换为**微秒级 (Microsecond) 整型**，并在处理 Trades 时默认原数据包含 8 小时偏移量。
